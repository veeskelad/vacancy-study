# vacancy-study

> 🌐 **Языки:** [English](README.md) · [Русский](README.ru.md)

[Claude Agent Skill](https://agentskills.io/specification), который превращает *«у меня собес через неделю»* в готовый персональный набор материалов для подготовки.

Скилл принимает **резюме** в любом формате (PDF, docx, markdown, текст, URL на LinkedIn / HH.ru / habr.com/career) и **описание вакансии**, после чего:

1. **Разбирает твой опыт** — задаёт уточняющие вопросы если чего-то не хватает
2. **Определяет тип роли** — DevOps, AI, Backend, Frontend, Data, Mobile или смешанная
3. **Сопоставляет вакансию с реальным опытом** — что сильная сторона, что пробел
4. **Создаёт hands-on лабы** под каждый пробел — теория + практика
5. **Адаптирует команды под твою ОС** — macOS, Linux (Debian/RHEL/Arch) или WSL
6. **Пишет prep-файл** — рассказ о себе на 2 минуты, маппинг «вакансия → опыт», топ-10 вопросов со структурами ответов, STAR+R истории, превентивные ответы на слабые места, 5 вопросов им
7. **Собирает шпаргалку теории и one-liner Q&A** — для утра собеса и боя за минуты до звонка

Работает в Claude Code, [Cowork](https://www.anthropic.com/news/cowork) и Claude.ai. Поддерживает любой язык — определяет язык твоего общения и отвечает на нём.

---

## Зачем это

Большинство подготовок к собесам уходят либо в общее («прочитай документацию по Kubernetes»), либо в тревожное («заучи 500 leetcode-задач»). По-настоящему помогает:

- **Честный gap-анализ** — понимание что в вакансии есть, а в резюме нет
- **Hands-on по пробелам** — чтобы можно было честно сказать *«вчера прогонял лабу по Kaniko — расскажу trade-offs»* вместо блефа
- **Отрепетированный рассказ о себе** — рассказ на 2 минуты это первое впечатление, его стоит шлифовать
- **Короткая шпаргалка** — чтобы скользить глазами утром перед звонком

Скилл кодирует этот workflow целиком. Тон — «рабочая тетрадь практика», а не «учебник для буткемпа».

## Установка

### Claude Code (через plugin marketplace, рекомендуется)

```
/plugin marketplace add veeskelad/vacancy-study
/plugin install vacancy-study@vacancy-study
```

После установки скилл активируется автоматически на фразах *«подготовь меня к собесу»* или явным вызовом `/vacancy-study`.

### Claude Code (ручная установка)

```bash
git clone https://github.com/veeskelad/vacancy-study.git
cp -r vacancy-study/skills/vacancy-study ~/.claude/skills/
```

Перезапусти Claude Code (или `/skills reload`). Скилл будет доступен во всех проектах.

Для project-scoped установки:

```bash
cp -r vacancy-study/skills/vacancy-study .claude/skills/
```

### Claude.ai

1. Склонируй или скачай этот репозиторий
2. Упакуй скилл: `python -m scripts.package_skill skills/vacancy-study`
3. В Claude.ai → **Settings → Capabilities → Skills → Upload** → выбери получившийся `.skill` файл

### Cowork

То же что ручная установка в Claude Code. Скопируй `skills/vacancy-study/` в Cowork-workspace, дальше вызывай скилл из чата.

## Быстрый старт

После установки просто вставь описание вакансии и резюме:

```
Вот вакансия Senior DevOps в [компания]. Описание прикладываю.
Резюме в cv.md. Готовь меня к собесу во вторник.
```

Или пошагово:

```
/vacancy-study
```

Скилл:

1. Один раз спросит про ОС (запомнит на будущее)
2. Поищет `cv.md` / `resume.md` / `profile.md` в проекте. Если не найдёт — попросит прислать резюме любым удобным способом.
3. Задаст 3–5 уточняющих вопросов если чего-то важного не хватает
4. Попросит описание вакансии если не прислал заранее
5. Сгенерирует всё

Что окажется на диске:

```
devops-labs/              # или ai-labs/, backend-labs/, и т.д. — под твою роль
├── theory/
│   └── NN-newtopic.md     # 5 секций теории на каждый gap
├── labs/
│   └── labNN-newtopic/
│       ├── README.md      # hands-on шаги под твою платформу
│       └── ...            # Dockerfile, манифесты, скрипты
├── THEORY-CHEATSHEET.md   # сжатая теория по всем темам — для утра
└── QUICKQA.md             # one-liner Q&A — для минут до звонка

interview-prep/
└── <vacancy-slug>-<YYYY-MM-DD>.md   # сам prep-файл
```

## Принципы дизайна

1. **Резюме — фундамент.** Без понимания опыта скилл превращается в генератор шаблонов. Шаг 1 обязателен; скилл будет задавать уточняющие вопросы, а не выдумывать.

2. **Универсальность.** Не предполагать конкретных имён файлов или структуры проекта. Проверять и адаптироваться к тому что есть.

3. **Не дублировать.** Если тема уже есть в проекте (под `devops-labs/`, `ai-labs/` и т.п.) — переиспользовать. Создавать только недостающее.

4. **Команды должны работать.** Каждая лаба адаптирована под платформу пользователя — `brew install` для macOS, `apt install` для Debian/Ubuntu, `dnf install` для RHEL/Fedora. Никаких сбоев на копипасте.

5. **Полнота теории.** Каждый theory-файл = 5 секций (что/зачем, ключевые понятия, архитектура, trade-offs, production-грабли). Тон — «личная заметка», не маркетинг.

6. **Рассказ о себе должен покрывать полный цикл работы** — архитектура → разработка → тесты → деплой → observability. И никогда не выдумывать опыт.

7. **Честность в пробелах.** Превентивный ответ типа *«в проде не использовал, вчера прогонял лабу — могу рассказать концепцию и trade-offs»* всегда сильнее блефа.

## Поддержка языков

Скилл определяет язык твоего первого сообщения и **отвечает на нём же**. Протестировано на английском, русском, немецком. Другие языки должны работать автоматически — Claude мультиязычный.

Внутри сгенерированных артефактов (theory, лабы, prep-файл) язык по умолчанию совпадает с вакансией + резюме. Если описание вакансии на русском, а резюме на английском — скилл спросит на каком языке делать output.

Код, имена файлов и технические идентификаторы всегда остаются в оригинале — никакой транслитерации.

## Чем этот скилл НЕ является

- **Не система скоринга вакансий.** Не говорит откликаться ли и насколько подходит роль. Для этого есть отдельные инструменты вроде [career-ops](https://github.com/santifer/career-ops).
- **Не leetcode-тренажёр.** Генерирует infrastructure/system лабы, не алгоритмические задачи.
- **Не CV-writer.** Читает твоё резюме; не переписывает и не полирует его.
- **Не auto-apply.** Никогда сам ничего не отправляет от твоего имени.
